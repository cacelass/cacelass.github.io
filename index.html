about Engineering identity
Approach

I approach ML as a systems discipline. A model only has value if it can be reproduced, validated, and deployed consistently. Everything else is experimentation, not engineering.

Systems background

ASIR-trained, with strong focus on systems administration and databases. Work includes schema design, query optimisation, indexing strategies, and understanding execution plans under real load.

Production mindset

I design pipelines that are resilient to schema changes, models that output calibrated probabilities instead of raw scores, and evaluation frameworks that explicitly prevent data leakage before deployment.

Tooling philosophy

Tools are not ideology. I choose based on constraints: simplicity when possible, complexity when necessary. Gradient boosting, linear models, or deep learning — depends on the problem, not preference.

projects Selected systems
dskit

Reproducible ML project scaffold designed to eliminate environment drift and enforce consistent structure across projects. Built to solve one of the most common failure points in ML: lack of standardisation.

credit-risk-classifier

Credit risk system focused on interpretability and decision usability. Uses calibrated probabilities, explicit thresholding, and evaluation aligned with business constraints rather than pure model metrics. AUC 0.81.

retainml

Customer churn prediction system that outputs ranked intervention lists instead of binary labels. Designed for operational decision-making, separating model output from business logic.

Stock-Market-Prediction

Time series prediction system built under strict temporal validation constraints. No random splits, no leakage, and evaluation aligned with real deployment conditions. Marginal improvement over baseline — expected in noisy markets.
